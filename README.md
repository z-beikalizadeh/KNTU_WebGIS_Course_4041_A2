مرین دوم WebGIS
درس: WebGIS
عنوان تمرین: پیاده‌سازی نقشه تعاملی با استفاده از OpenLayers
________________________________________
معرفی پروژه
در این پروژه یک سامانه ساده WebGIS طراحی شده است که امکان نمایش نقشه، جستجوی مکانی و دریافت اطلاعات توصیفی از نقاط مختلف را فراهم می‌کند.
هدف اصلی این تمرین آشنایی عملی با کتابخانه OpenLayers و نحوه ارتباط با سرویس‌های وب از طریق Fetch API در زبان JavaScript است.
________________________________________
قابلیت‌های پروژه
این سامانه دارای قابلیت‌های زیر می‌باشد:
نمایش نقشه پایه شهر تهران
جستجوی مکانی و نمایش مکان موردنظر با یک نشانگر روی نقشه
امکان کلیک روی نقشه و دریافت آدرس تقریبی آن نقطه
طراحی ساده و قابل استفاده در موبایل و دسکتاپ
________________________________________
ابزارها و فناوری‌های استفاده‌شده
در پیاده‌سازی این پروژه از ابزارها و فناوری‌های زیر استفاده شده است:
OpenLayers برای نمایش و مدیریت نقشه
API نشان برای انجام جستجوی مکانی در محدوده ایران
OpenStreetMap به‌عنوان نقشه پایه و برای آدرس‌یابی معکوس
Fetch API برای دریافت داده‌ها به‌صورت غیرهمزمان
HTML و CSS برای ساختار و طراحی ظاهری صفحه
________________________________________
مقایسه APIهای ژئوکدینگ
در مرحله انتخاب API ژئوکدینگ، چند سرویس مختلف بررسی شد.
سرویس Google Geocoding به دلیل نیاز به پرداخت هزینه و محدودیت‌های دسترسی گزینه مناسبی نبود.
API نشان به دلیل دقت بالا برای مکان‌های داخل ایران و دسترسی مناسب انتخاب شد.
همچنین از سرویس Nominatim مربوط به OpenStreetMap به‌عنوان گزینه عمومی برای برخی درخواست‌ها استفاده شده است.
با توجه به شرایط دسترسی در ایران، API نشان انتخاب نهایی این پروژه بوده است.
________________________________________
مقایسه APIهای هواشناسی
برای بخش داده‌های هواشناسی، سرویس‌های مختلف بررسی شدند.
سرویس OpenWeatherMap نیاز به کلید API دارد و در نسخه رایگان دارای محدودیت است.
در مقابل، سرویس Open-Meteo بدون نیاز به کلید API قابل استفاده است.
در این پروژه به‌دلیل تمرکز بر مفاهیم WebGIS و محدودیت دسترسی، از داده‌های نمایشی و ساده‌سازی‌شده استفاده شده است.
